# 修改方案：七星彩适配 2020 年新规则

> 日期：2026-06-07
> 问题：代码全部按旧规则（7位0-9）实现，未适配2020年新规则（前区6位0-9 + 后区1位0-14）

---

## 1. 新旧规则对比

| | 旧规则（2020年前） | 新规则（2020年后） |
|------|---------|---------|
| 前区 | 7位，每位 0-9，可重复 | 6位，每位 0-9，可重复 |
| 后区 | 无 | 1位，0-14 |
| 总组合数 | 10⁷ = 1,000万 | 10⁶ × 15 = 1,500万 |

---

## 2. 前置确认：数据源格式

**必须先确认** `data.17500.cn/7xc_asc.txt` 返回的实际格式是什么。

可能情况：

### 情况A：数据源已是新规则

TXT 格式示例：
```
25001 2025-01-03 5 2 8 1 4 7 11 ...
```
前6位(0-9) + 第7位(0-14) + 销售额 + 奖池...

→ 按方案A修改

### 情况B：数据源仍是旧规则

TXT 格式示例：
```
25001 2025-01-03 5 2 8 1 4 7 3 ...
```
7位全是0-9

→ 需要评估：是切换数据源，还是保持旧规则兼容（方案B）

---

**改为**：
```sql
CREATE TABLE IF NOT EXISTS lottery_7xc (
    ...
    num_1 TINYINT NOT NULL COMMENT '前区1(0-9)',
    num_2 TINYINT NOT NULL COMMENT '前区2(0-9)',
```

**变更点**：
- `num_7` → `back_1`（后区）

- 字段注释更新

> ⚠️ **注意**：如果数据库已有数据，字段重命名需要用 ALTER TABLE，或先备份再重建。

### 3.2 `app/cleaner/qxc_cleaner.py` — 修改解析逻辑

**当前**（第30行）：
```python
nums = [int(p) for p in parts[2:9]]
# ...
# 七星彩每位 0-9
if any(n < 0 or n > 9 for n in nums):
    return None
```

**改为**：
```python
# 前6位 0-9
front_nums = [int(p) for p in parts[2:8]]
back_num = int(parts[8])  # 后区 0-14

# 前区校验：0-9
if any(n < 0 or n > 9 for n in front_nums):
    return None
# 后区校验：0-14
if back_num < 0 or back_num > 14:
    return None
```

**字段映射更新**：

```python
@property
def fields(self) -> list[str]:
```
