# 前端换肤方案：深色 → 浅色

> 状态：**实施完成（2026-06-07 20:55）**
> 目标：将整套前端界面从深色背景（#0d1117 系）切换为浅色背景（#ffffff / #f5f7fa 系），并对所有色板做系统性适配。
> 范围：仅前端（`frontend/` 目录下），不涉及后端 API、数据库、爬虫、统计计算。

## 0. 用户额外需求（2026-06-07 20:39）

1. **双主题可切换**（A 方案）：顶部加切换按钮，深/浅模式 + localStorage 记忆 ✅
2. **系统标题后加免责声明**（小字、低对比度，提示数据仅供统计分析） ✅
3. **表格每页条数默认 10**，可切换 10/20/50（当前默认 20，需调整） ✅

## 实施完成项

- ✅ ECharts `light` 主题完整色板（替换占位 = dark）
- ✅ `themeColor(dark, light)` 工厂函数

**核心挑战：**
1. CSS 用了大量 `rgba(255,255,255,0.xx)` 作为"在深色背景上的高光/边框"，浅色背景上会变成"白上加白"不可见
2. ECharts `dark` 主题写死了一套暗色配色，需新建 `light` 主题
3. 已嵌入的固定颜色（按钮、号码球、数据系列颜色）需要做"主题无关"重构

## 2. 实施策略

### 2.1 三选一策略（建议选 A）

| 策略 | 描述 | 优 | 缺 |
|------|------|----|---|
| **A. 引入主题切换（推荐）** | CSS 变量 + `data-theme` 属性 + ECharts `light`/`dark` 双主题 | 用户可切换；改动一次兼容两套 | 工作量略大 |
| **B. 直接全量替换为浅色** | 把所有 dark 颜色硬编码替换为 light | 简单直接 | 失去深色模式，以后想加回需重做 |
| **C. CSS 变量 + 单套硬编码** | 用 CSS 变量统一外观，但 ECharts 仍写死 light | 折中 | ECharts 部分仍耦合 |

> **我推荐 A**：用户后续很可能想保留深色选项（深色更护眼），A 方案一次到位。

## 5. 风险与回退

### 风险
1. **遗漏硬编码色**：CSS/JS 中散落的 hex 值可能漏改，表现为"白底黑字控件上还残留深色块"
2. **ECharts 主题缓存**：已渲染的图表不会自动换主题，必须 dispose + 重渲染
3. **localStorage 残留**：旧用户本地存的 `theme=dark` 字段可能导致默认行为变化

### 回退
- 全部改动均在 frontend/ 目录，未触及后端
- 单一 git commit 可整体回退
- 暂不提供"用户偏好跟随系统"功能（如 `prefers-color-scheme`），等浅色稳定后再加

### 测试覆盖
- 5 彩种 × 3 视图 = 15 个图表类型
- 切换主题后所有 15 个图表需重新验证：文字可读、对比度、坐标轴、图例、tooltip、dataZoom、visualMap、热力图色阶
