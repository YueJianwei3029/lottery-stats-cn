## 变更一：`GET /api/{彩种}/stats/advanced/normal` — 返回格式变更

### 影响范围
- 前端文件：`frontend/js/app.js` → `loadAdvNormal()` 函数
- 涉及彩种：全部 5 个彩种，但仅 DLT 和 SSQ 有实际数据

### 变更原因
原实现按"位置"逐字段统计（如大乐透前区1~前区5各出一张图），统计学意义有限。
新方案改为按"号码值"跨位置聚合（如大乐透前区 5 个位置合并统计每个号码 1-35 的总出现频次），利用中心极限定理做正态分布拟合，更有统计意义。

### 旧格式

```json
{
  "fields": ["front_1", "front_2", "front_3", "front_4", "front_5", "back_1", "back_2"],
  "stats": [
    {
      "field": "front_1",
      "mean": 18.23,
      "std": 10.15,
      "skewness": -0.12,
      "kurtosis": -1.05,
      "histogram": {"bins": [2, 5, 8, ...], "counts": [120, 200, ...]}
    },
    // ... 每个字段一条
  ]
}
```

### 新格式

```json
{
  "type": "dlt_front",           // "dlt_front" | "ssq_red" | null
  "num_range": [1, 35],          // 号码范围 [min, max]
  "total_periods": 2880,         // 统计期数
  "total_draws": 14400,          // 总抽取次数 (期数 × 每期位数)
  "expected_freq": 411.43,       // 理论期望频次
  "expected_std": 20.07,         // 理论标准差
  "skewness": 0.15,              // 实际频次分布偏度
  "kurtosis": -0.32,             // 实际频次分布峰度
  "histogram": {
    "numbers": [1, 2, 3, ..., 35],     // X轴：号码值
    "counts": [410, 398, 415, ...]     // Y轴：各号码出现总频次
  },
  "theoretical_curve": {
    "x": [380, 382, 384, ...],         // 理论正态曲线X轴采样点（频次范围）
    "y": [0.010, 0.015, 0.020, ...]    // 理论正态曲线Y轴（概率密度×缩放）
  }
}
```

### 各彩种返回对照

| 彩种 | `type` 值 | `num_range` | `histogram.numbers` 数量 |
|------|----------|-------------|------------------------|
| 超级大乐透 | `"dlt_front"` | `[1, 35]` | 35 个 |
| 双色球 | `"ssq_red"` | `[1, 33]` | 33 个 |
| 七星彩 | `null` | — | — |
| 排列5 | `null` | — | — |
| 排列3 | `null` | — | — |

### 前端适配要点

1. **`type === null`**：显示占位提示 "当前彩种不支持正态分布分析"
2. **`type === "dlt_front"` 或 `"ssq_red"`**：渲染 **1 张图**（原先是 N 张）
   - 柱状图：`histogram.numbers` → X轴，`histogram.counts` → Y轴
   - 理论正态曲线（虚线）：用 `theoretical_curve.x` / `theoretical_curve.y` 绘制，需映射到号码X轴
   - 实际拟合曲线（实线）：前端用 `histogram.counts` 的 mean/std 自行计算正态曲线
3. 图表底部显示统计量：`μ(理论)=expected_freq σ(理论)=expected_std 偏度=skewness 峰度=kurtosis`

## 变更二：七星彩字段名变更（待数据源确认后执行）

### 影响范围
- 前端文件：`frontend/js/app.js` — 表格渲染、图表渲染中涉及七星彩字段名的地方
- 涉及彩种：仅七星彩

### 变更原因
七星彩 2020 年规则变更：旧规则 7 位全 0-9 → 新规则 前区 6 位(0-9) + 后区 1 位(0-14)。
字段名需对应更新。

### 字段对照

| 旧字段名 | 新字段名 | 含义 | 范围 |
|---------|---------|------|------|
| `num_1` | `num_1` | 前区1 | 0-9 |
| `num_2` | `num_2` | 前区2 | 0-9 |
| `num_3` | `num_3` | 前区3 | 0-9 |
| `num_4` | `num_4` | 前区4 | 0-9 |
| `num_5` | `num_5` | 前区5 | 0-9 |
| `num_6` | `num_6` | 前区6 | 0-9 |
| `num_7` | `back_1` | **后区** | **0-14** |

### 前端适配要点

1. **表格渲染** (`renderTable()`)：`FIELD_LABELS` 中七星彩的标签
   - `num_7: "号码7"` → `back_1: "后区"`
2. **号码球颜色** (`getBallClass()`)：七星彩 `num_*` 目前是 `ball-gray`，需确认后区 `back_1` 是否用不同颜色
3. **位置标签** (`buildPosLabels()`, `buildHeatmapLabels()`)：
   - 旧：`["号码1","号码2","号码3","号码4","号码5","号码6","号码7"]`
   - 新：`["前区1","前区2","前区3","前区4","前区5","前区6","后区"]`

### 执行时机
⚠️ 此变更需等后端确认数据源格式后同步执行，目前暂不修改。前端可先按新字段名预留兼容。
