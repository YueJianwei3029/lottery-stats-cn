## 2. 详细改动记录

### 2.1 项 1：Crawler 工厂合并

**改动前**：
```
app/crawler/
├── __init__.py
├── base_crawler.py
├── pl3_crawler.py      (16 行)
├── ssq_crawler.py      (16 行)
├── pl5_crawler.py      (16 行)
├── crawler_7xc.py      (16 行)
└── dlt_crawler.py      (16 行)
```

**改动后**：
```
app/crawler/
├── __init__.py         (更新导入路径)
├── base_crawler.py
└── factory.py          (新增, 73 行, 集中管理 5 个类)
```

**实现要点**：
- 新建 `app/crawler/factory.py`，提供 `make_crawler(table_key)` 工厂函数
- 工厂函数用 `type()` 动态创建 `BaseCrawler` 子类，类名按 `lottery_xxx` 推导
- 对外暴露原有 5 个类名（`Pl3Crawler` / `SsqCrawler` / `Pl5Crawler` / `QxcCrawler` / `DltCrawler`），**调用方代码零修改**
- 历史类名兼容：`7xc → QxcCrawler`（短名映射表）

**收益**：
- 文件数：6 → 3（-50%）
- 模板代码：~80 行 → 0（消除重复）
- 新增彩种：从"复制整个文件"→"在 `__init__.py` 注册表加一行"

### 2.3 项 6：删除 `has_normal_advanced` 字段

**位置**：`app/core/router_factory.py`

**改动**：
- 删除 `LotteryConfig` 中的 `has_normal_advanced: bool = False` 字段
- 删除 dlt/ssq 配置中的 `has_normal_advanced=True` 赋值

**原因**：该字段在 `RouterFactory.build` 中**无任何分支逻辑读取**，对所有彩种都开放 `/stats/advanced/normal` 路由。属于"配置项形同虚设"。
