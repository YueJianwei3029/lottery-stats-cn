## 3. 开发顺序建议

```
Phase 1 (基础设施)
  └→ Phase 2 (数据采集)
       └→ Phase 3 (统计服务)
            └→ Phase 4 (API路由)
                 └→ Phase 5 (前端)
                      └→ Phase 6 (Docker)
                           └→ Phase 7 (验证)
```

## 4. 关键注意事项

- **编码**：所有文件 UTF-8 无 BOM
- **PowerShell**：不支持 `&&` 串联，使用 `;` 或 `foreach`
- **numpy**：需在 `requirements.txt` 中声明 `numpy>=1.26.0`
- **API 统一格式**：所有端点返回 `{code: 200, message: "success", data: {...}}`
- **扩展统计端点**：统一为 `position / hot_cold / period_list / ratio`（4项）
- **高级统计端点**：统一为 `advanced/std / advanced/percentile / advanced/normal`（3项）
- **config.js**：必须在 `app.js` 之前加载
- **Docker**：nginx 镜像使用 `nginx:alpine`，配置挂载为只读
