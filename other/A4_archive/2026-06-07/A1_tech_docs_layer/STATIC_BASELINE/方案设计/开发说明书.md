## 2. 核心业务流程（step by step）

| 步骤 | 触发条件 | 执行内容 | 产出结果 |
|------|---------|---------|---------|
| 1. 服务初始化 | 首次启动服务 | 自动连接 MySQL，创建数据库 `lottery_db`，按 5 个彩种分别建表 | 数据库 + 5 张空表就绪 |
| 2. 定时爬取 | 每 24h 定时触发 | 逐一访问 5 个彩种的公开 TXT 数据源，下载最新开奖数据 | 原始 TXT 文本数据 |
| 3. 数据清洗 | 爬取完成后 | 解析 TXT 格式，提取期号/日期/号码（按官方规则拆分到独立字段），校验有效性 | 结构化清洗数据 |
| 4. 增量入库 | 清洗完成后 | 按期号比对，仅插入新期号数据（INSERT IGNORE） | 数据库新增记录（无重复） |
| 5. 数据查询 | 用户访问前端 | 接收 date/end_date/page 参数，从数据库检索最新置顶返回 JSON | 分页查询结果 |
| 6. 基础统计 | 用户查看统计 | 调用 numpy 向量化统计 → 号码频率/奇偶比/大小比（按位置拆分） | 统计指标 JSON |
| 7. 扩展统计 | 用户切换扩展统计 Tab | 调用对应彩种 4 个扩展端点（position/hot_cold/period_list/ratio） | 深度分析数据 |
| 8. 前端渲染 | 收到接口数据 | ECharts 绑定数据，渲染柱状图/饼图/热力图/折线图等，支持 dataZoom 滑块 | 可视化图表页面 |

## 5. 边界条件与异常处理

| 异常情况 | 处理策略 |
|---------|---------|
| 数据库连接失败 | 记录错误日志，本次采集跳过，下次定时重试 |
| 外部数据源不可访问 | 超时 30s 后放弃，记录日志，不阻塞其他彩种 |
| TXT 数据格式变更 | 清洗模块校验失败则跳过该行，记录警告日志 |
| 期号重复 | UNIQUE 约束自动跳过，INSERT IGNORE 策略 |
| 源数据缺失字段 | 对应字段置空/NULL，接口正常返回 |
| 分页参数越界 | 返回空列表，不报错 |
| 日期范围无数据 | 返回空列表，附带提示信息 |
| 统计计算异常 | 返回 `{code: 500, message: error}` |
