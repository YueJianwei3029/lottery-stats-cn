> **定位**：业务需求与目标，是开发与验收的依据。
> **原则**：只定义"做什么"和"为什么做"，不涉及"怎么做"。
> **约束**：不得包含代码实现、API 参数、数据库字段。

---

## 1. 项目背景

- **项目名称**：彩票数据统计与可视化系统
- **业务目标**：对 5 类公开彩票文本数据完成自动化爬取、清洗、结构化存储，搭建标准化统计接口，通过纯 HTML 前端实现数据可视化
- **目标用户**：内部数据分析人员
- **核心价值**：实现彩票数据的自动化管理与多维度分析，无需人工干预
- **部署模式**：本地 Windows 开发 + Docker 云端部署（www.yuejw.top）

## 3. 非功能需求

- **自动化**：每 24 小时自动爬取 1 次，无需人工干预
- **数据完整性**：按期号增量去重，UNIQUE 约束保证数据不重复
- **数据规范**：严格遵循三范式，号码拆分独立字段，无逗号拼接、无冗余
- **性能**：numpy 向量化加速，单次统计 < 5ms；散点/LLN 500 期 < 100ms
- **兼容**：API 返回格式保持统一 `{code, message, data}`
- **部署**：本地 `start.bat`，云端 `docker compose up -d`

## 4. 边界与约束

- **不做什么**：
  - 不提供用户登录/权限系统
  - 不做实时推送/WebSocket
  - 不做数据导出（Excel/PDF）
  - 不引入机器学习/预测模型
  - 七乐彩销量/奖池若源数据缺失，仅做接口占位
- **技术约束**：
  - Python 3.13 + MySQL 8.0 + FastAPI + 纯 HTML/JS/ECharts + numpy
  - 路由层 RouterFactory 配置驱动模式
