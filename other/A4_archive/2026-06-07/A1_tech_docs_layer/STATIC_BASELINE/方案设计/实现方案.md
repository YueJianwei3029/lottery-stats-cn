## 1. 模块概述

| 模块 | 职责 | 核心设计 |
|------|------|---------|
| core/database | 建库建表、连接池、CRUD | 单例 Database 类，提供 init_db() / query() / fetch_all() |
| core/scheduler | 每 24h 触发采集 | APScheduler BackgroundScheduler，startup 时注册 + 立即执行 |
| core/config | 集中管理配置 | 环境变量优先，支持 Docker/本地切换 |
| crawler | 下载 TXT 数据 | 基类 BaseCrawler + 5 个子类，子类仅覆盖 url/encoding |
| cleaner | 解析 TXT → 结构化 | 基类 BaseCleaner + 5 个子类，子类定义 fields 映射 |
| routers | API 端点 | 每个彩种一个路由文件，统一调用 services |
| services | 统计计算 | numpy 向量化加速，纯函数设计 |
| frontend | 纯 HTML 前端 | LotteryApp 类，单页 4 Tab 切换 + ECharts 渲染 |

## 2. 核心流程

### 2.1 服务启动流程

```
main.py 启动
  → database.init_db() 建库建表（IF NOT EXISTS）
  → scheduler.start() 注册 24h 定时 + 立即执行首次采集
  → 本地模式：mount /frontend StaticFiles
  → uvicorn 启动 API 服务
  → 暴露 /api/health 健康检查端点
```

### 2.2 数据采集流水线

```
scheduler 触发（每24h或启动时）
  → 遍历 5 个彩种配置（从 config.DATA_SOURCES）
    → crawler.fetch(url, encoding) 下载 TXT 文本
    → cleaner.parse(txt) 返回 List[dict]
    → database.batch_insert(table, records)
      → 逐条 INSERT IGNORE（按期号 draw_num UNIQUE 去重）
    → 记录日志（新增条数/跳过条数）
```

### 2.3 API 请求流程

```
GET /api/lottery_dlt/query?date=20260101&end_date=20260601&page=1&page_size=20
  → router 校验参数（date格式、page>=1、page_size 1-100）
  → database.query(table, date, end_date, page, page_size)
    → SELECT * FROM table
      WHERE draw_date >= date AND draw_date <= end_date
      ORDER BY draw_date DESC
      LIMIT page_size OFFSET (page-1)*page_size
    → SELECT COUNT(*) FROM table WHERE ...  （total）
  → 返回统一格式 {code, message, data: {total, page, page_size, records}}
```

### 2.4 统计计算流程（numpy）

```
GET /api/lottery_dlt/stats
  → router 解析 date/end_date 参数
  → database.fetch_all(table, date, end_date) 获取 records
  → services.base_stats(table, records, date, end_date)
    → 提取二维数组 arr = np.array([[r[f] for f in fields] for r in records])
    → 频率：np.unique(arr[:, i], return_counts=True)
    → 奇偶：np.sum(col % 2 == 1) / np.sum(col % 2 == 0)
    → 大小：np.sum(col >= threshold) / np.sum(col < threshold)
  → 返回计算结果 JSON
```

## 4. 关键算法（numpy 向量化）

| 算法 | numpy 实现 | 复杂度 |
|------|-----------|--------|
| 号码频率 | `np.unique(arr[:, i], return_counts=True)` | O(n) |
| 奇偶比 | `np.sum(col % 2 == 1)` / `np.sum(col % 2 == 0)` | O(n) |
| 大小比 | `np.sum(col >= threshold)` / `np.sum(col < threshold)` | O(n) |
| 和值 | `np.sum(arr[:, :fields_n], axis=1)` | O(n) |
| 跨度 | `np.max(fronts, axis=1) - np.min(fronts, axis=1)` | O(n) |
| 区间分布 | `np.histogram(all_fronts, bins=zones)` | O(n) |
| 标准差 | `np.std(col)` | O(n) |
| 分位数 | `np.percentile(col, [25, 50, 75])` | O(n log n) |
| 偏度 | `np.mean(((x - mean) / std) ** 3)` | O(n) |
| 峰度 | `np.mean(((x - mean) / std) ** 4) - 3` | O(n) |
| 冷热号 | 按频率排序取 top/bottom N | O(n log n) |
