> **版本**：v2.0 | **日期**：2026-06-07

## 1. 环境准备

```bash
# Python 3.13
pip install -r requirements.txt

# MySQL 8.0（本地或 Docker）
docker run -d --name lottery_mysql -p 3306:3306 -e MYSQL_ROOT_PASSWORD=123456 mysql:8.0
```

## 2. 本地开发

```bash
# 方式一：双击 start.bat
# 方式二：手动
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000

# 访问 http://localhost:8000/frontend/index.html
# API 文档 http://localhost:8000/docs
```

## 3. Docker 部署

```bash
echo "DB_PASSWORD=lottery123" > .env
docker compose down && docker compose up -d
# 等待首次数据同步（2-3 分钟）
docker compose logs -f app
# 访问 http://localhost
```

## 4. 代码规范

### 4.1 文件组织
- **不可修改**：`A0_entry_layer/`（AI 无权修改）
- **静态基线**：`A1/STATIC_BASELINE/`（需用户确认）
- **动态工作**：`A1/DYNAMIC_WORK/` + `A3_scratch/`（AI 自由操作）

### 4.2 编码规范
- UTF-8 无 BOM
- Python 2 空格缩进（非必须，但推荐）
- JS 无严格规范（当前混合 var/let/const，渐进式优化）

### 4.3 命名约定
- 后端字段：小写蛇形（snake_case）
- API 端点：小写蛇形
- 前端变量：驼峰（camelCase），`currentType` `currentTheme` 等全局状态
- 彩种标识：`lottery_dlt` / `lottery_7xc` / `lottery_ssq` / `lottery_pl5` / `lottery_pl3`
- 图表 dom id：`chartFreq` / `extHeat` / `adv-scatter-{name}` 等

## 6. 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 启动后无数据 | 首次爬取需要时间 | 等待 2-3 分钟或查看日志 |
| 图表无数据 | 后端字段顺序与前端 MAP 不一致 | 检查 `buildPosName` / `buildHeatmapLabels` MAPS |
| 七星彩后区全 0 | 数据库 `back_1` 列无数据 | 检查清洗器是否正确解析 TXT 后区字段 |
| 散点图标题为"第X位" | 后端返回原始名 | 前端 `buildPosName()` 自动映射，确认 MAPS 顺序 |
| 主题切换后图表异常 | ECharts 实例未销毁 | 切换时调用 `disposeCharts()` + 重渲染 |
