# 实现方案

> **版本**：v2.0 | **日期**：2026-06-07

## 1. 模块实现要点

### 1.1 `app/core/`

| 文件 | 关键函数 | 实现注意事项 |
|------|---------|-------------|
| `config.py` | — | 环境变量覆盖（`os.getenv`），`DATA_SOURCES` dict 驱动爬虫工厂 |
| `database.py` | `db.init_database()` / `db.query()` / `db.batch_insert()` | `init_database` 建库建表用 CREATE TABLE IF NOT EXISTS；批量插入 INSERT IGNORE 去重；分页查询用 LIMIT/OFFSET |
| `numpy_utils.py` | `records_to_array(records, fields)` | 返回 float64 ndarray，缺失值 NaN |
| `router_factory.py` | `RouterFactory.build(config)` | 动态生成路由函数，注意闭包变量：每个路由内部用 `ext = importlib.import_module(config.ext_module)` 避免循环引用 |
| `scheduler.py` | `start_scheduler()` / `stop_scheduler()` | 启动时全量 + 后续每 24h 增量；APScheduler BackgroundScheduler |

### 1.2 `app/crawler/`

| 文件 | 说明 |
|------|------|
| `base_crawler.py` | `BaseCrawler.fetch(url)` → 返回 `response.text` |
| `factory.py` | `create_crawler(url, encoding)` → 动态生成继承 BaseCrawler 的子类 |
| `__init__.py` | `LOTTERY_REGISTRY` dict + `run_all_crawlers()` → 遍历注册表，fetch→parse→batch_insert，间隔 3s 防限流 |

### 1.3 `app/cleaner/`

| 文件 | 解析规则 |
|------|---------|
| `base_cleaner.py` | 逐行 split，跳过日期行 |
| `pl3_cleaner.py` | 号码 × 1 行 → num_1/num_2/num_3（0-9） |
| `pl5_cleaner.py` | 号码 × 1 行 → num_1~num_5（0-9） |
| `ssq_cleaner.py` | 红球 6 位 + 蓝球 1 位，范围校验 |
| `dlt_cleaner.py` | 前区 5 位(1-35) + 后区 2 位(1-12) |
| `qxc_cleaner.py` | 前区 6 位(0-9) + 后区(0-14)，额外处理销售/奖池行 |

### 1.4 `app/services/`

| 文件 | 函数 | 算法 |
|------|------|------|
| `base_stats.py` | `base_stats(table, date, end_date)` | numpy unique + bincount：频率；num % 2 统计奇偶；≥threshold 统计大小 |
| `dlt_stats.py` | `position_stats/hot_cold_stats/period_list_stats/ratio_stats` | 前区 1-35 后区 1-12，numpy unique + 排序 |
| `ssq_stats.py` | 同上 | 红球 1-33 蓝球 1-16 |
| `qxc_stats.py` | 同上 + `digit_freq_stats/trend_stats` | **前区6位+后区分开处理**；冷热号返回 `{front:{hot,cold}, back:{hot,cold}}`；热力图用 `pos_1~pos_7` |
| `pos_stats.py` | 同上（`table, fields` 参数化） | PL3/PL5 共用，阈值和范围按参数动态 |
| `advanced_stats.py` | `scatter_data/lln_data/std_dev_stats/percentile_stats` | 散点：每位置逐期取值；LLN：逐期计算运行均值+标准差+±1σ/±2σ 置信带；`TABLE_FIELDS` dict 定义字段顺序 |

### 1.5 `app/main.py`

- lifespan 中 `db.init_database()` + `start_scheduler()`
- CORS 全放开
- RouterFactory 批量注册
- StaticFiles 挂载 frontend

### 2.2 ECharts 双主题

```js
var currentTheme = localStorage.getItem("lottery-theme") || "dark";
var THEME_ECHARTS = { dark: "dark", light: null };

function initChart(dom) {
    return echarts.init(dom, THEME_ECHARTS[currentTheme]);
}
function themeColor(dark, light) {
    return currentTheme === "light" ? light : dark;
}
```

切换主题时重新注册 ECharts light 主题，dispose 全部实例后重新渲染。

## 3. 命名约定

### 3.1 位置标签统一

所有前端图表中使用的位置名称必须与表格表头一致：

| 彩种 | 字段 → 标签示例 |
|------|---------------|
| 七星彩 | num_1~6 → "前1"~"前6"，back_1 → "后区" |
| 排列5 | num_1~5 → "万位""千位""百位""十位""个位" |
| 排列3 | num_1~3 → "百位""十位""个位" |
| 大乐透 | front_1~5 → "前区1"~"前区5"，back_1~2 → "后区1""后区2" |
| 双色球 | red_1~6 → "红球1"~"红球6"，blue_1 → "蓝球" |

实现：
- `buildNumLabels()` — 表格表头
- `buildPosLabels(fields)` — 基础统计图例
- `buildHeatmapLabels(positions, lotType)` — 热力图 y 轴
- `buildPosName(index, lotType)` — 散点图/LLN 标题

### 3.2 后端字段

- 七星彩：`num_1~num_6` + `back_1`（非 num_7）
- 大乐透：`front_1~5` + `back_1~2`
- 双色球：`red_1~6` + `blue_1`
- PL5/PL3：`num_1~N`

## 4. 部署

- **本地**：`pip install -r requirements.txt` → `start.bat`（uvicorn app.main:app）
- **Docker**：`docker compose up -d`，3 容器启动后 app 自动建库 + 首次爬取
- **环境变量**：`DB_HOST/PORT/USER/PASSWORD`，`.env` 文件或 shell 注入
