# 需求说明书

> **定位**：业务需求与目标，开发与验收的最终依据。
> **版本**：v2.0（Phase 12 完成）| **日期**：2026-06-07

---

## 1. 项目背景

- **项目名称**：彩票数据统计与可视化系统
- **业务目标**：对 5 类公开彩票数据完成自动化爬取、清洗、存储，搭建标准化统计 API，通过纯 HTML 前端实现可视化
- **目标用户**：数据分析人员
- **核心价值**：自动化管理与多维度分析，无需人工干预
- **部署模式**：本地 Windows 开发（`start.bat`）+ Docker 云端部署（`docker compose up -d`）

## 2. 彩票种类与规则

| 彩种 | 表名 | 号码字段 | 范围 |
|------|------|----------|------|
| 超级大乐透 | lottery_dlt | front_1~5(前区) + back_1~2(后区) | 前区1-35, 后区1-12 |
| 七星彩 | lottery_7xc | num_1~6(前区) + back_1(后区) | 前区0-9, 后区0-14 |
| 双色球 | lottery_ssq | red_1~6(红球) + blue_1(蓝球) | 红球1-33, 蓝球1-16 |
| 排列5 | lottery_pl5 | num_1~5 | 0-9 × 5 |
| 排列3 | lottery_pl3 | num_1~3 | 0-9 × 3 |

## 4. 非功能需求

- **自动化**：每 24h 自动爬取，启动时全量 + 后续增量
- **数据完整性**：draw_num UNIQUE 约束，INSERT IGNORE 增量去重
- **性能**：numpy 向量化，单次统计 < 5ms
- **兼容**：API 统一 `{code:200, message:"success", data:{...}}`
- **部署**：本地 `start.bat`，云端 `docker compose up -d`

## 5. 边界约束

- 不提供用户登录/权限系统
- 不做实时推送/WebSocket
- 不做数据导出（Excel/PDF）
- 不引入机器学习/预测模型
- 七乐彩未纳入
