## 文档体系说明

本项目采用文档驱动的开发模式（v1.5），所有文档在 `other/` 目录下按层级组织。

详细规则见 `other/A0_entry_layer/CONSTRAINT_LIST.md`。

### 项目等级自动判定

收到新项目需求后，AI 自动判定等级并告知用户：

| 等级 | 触发信号 | 文档配置 |
|------|----------|----------|
| **轻量** | "原型"、"demo"、"试一下"、≤2 功能、无前端 | 极简：需求 + 架构(合并) + 实现(合并) + 仅开发进度 |
| **标准** | "正式"、"上线"、≥5 功能、有数据库+前端、或需求模糊时默认 | 核心：完整 A1S 链 + 开发进度 + 踩坑记录 |
| **完整** | 多人协作、"并行开发"、"多个Agent" | 全量：含缺陷清单、变更日志、协作表、A2、A3 |

### AI 一键启动模式（新项目）

当用户输入 **"开始"、"start"、"启动"** 或类似指令时，AI 自动执行：

```
第一步：自动判定项目等级 → 告知用户配置 → 用户点头即开工
第二步：读取 A0_entry_layer/ → 了解项目背景和约束
第三步：读取 TEMPLATES/ → 获取必备文档模板结构
第四步：按等级在 A1/STATIC_BASELINE/ 下创建文档（数量按等级裁剪）：
   ① 需求说明书.md         （三级均必选）
   ② 开发说明书.md         （标准/完整必选，轻量合并到需求）
   ③ 方案设计/              （按等级裁剪）
       ├ 系统架构.md        （三级均必选，轻量合入数据+接口摘要）
       ├ 数据设计.md        （标准/完整必选）
       └ 接口设计.md        （标准/完整必选）
   ④ 实现方案.md            （轻量/标准单文件，完整按模块拆分）
   ⑤ 任务分解.md            （标准/完整必选，轻量合并到实现方案）
第五步：每完成一个文档，请求用户确认
第六步：全部确认后 → 初始化 DEV_INDEX → 等待用户指令进入开发
```

### 增量开发模式（已有项目加功能）

当 `START.md` 状态为"进行中"或"已完成"，用户追加新功能时：

```
铁律：没读完以下 5 份文档，不准输出任何方案或代码
   ① START.md          ② DEV_INDEX.md      ③ 需求说明书.md
   ④ 开发进度.md        ⑤ 系统架构.md

读完现状后，AI 自动分类需求：
   🔧 小改动 → 在现有文档中追加条目，不走设计循环
   📦 新功能 → 追加章节，必要时新建子设计文档
   🏗 大功能 → 完整 mini 设计循环，可选 A3_scratch 暂存

追加原则：追加不覆盖，新功能编号延续 F-NN
追加后检查是否需要升级项目等级
```

### 可自定义的启动模式

用户可要求：
- **简化模式**：AI 按默认模板自动推进，关键节点汇报
- **详细模式**：每个文档完成后都需要用户深度审阅和调整
- **指定文档**：仅创建指定的某几个文档

## Key Rules

- **讨论阶段不输出代码** — 用描述、逻辑、技术选型代替
- **禁止全局批量文本替换**（编码安全）
- **先输出方案/文档，再修改文件**，除非用户给出明确执行指令
- **Windows 环境** — 使用 PowerShell 语法
- **UTF-8（无 BOM）** 编码所有文件
- **ALL_PROMPTS.txt** — 每轮对话结束时自动追加用户输入。格式：`## [YYYY-MM-DD 第N轮]` + 逐行用户输入
- **轻量/标准级**：静态基准变更在文档底部加 `> [日期] 变更：xxx` 即可。**完整级**：先记 CHANGELOG，经审批再改。

## Key Files Reference

- `other/A0_entry_layer/START.md` — 项目入口，含项目等级和当前阶段
- `other/A0_entry_layer/CONSTRAINT_LIST.md` — AI 行为约束
- `other/A0_entry_layer/DEV_NOTES.txt` — 用户开发笔记（想法/痛点/规则），AI 只读
- `other/A1_tech_docs_layer/DEV_INDEX.md` — 文件索引
- `other/TEMPLATES/` — 文档模板
- `other/ALL_PROMPTS.txt` — 对话流水存档
