## 2026-06-07 前端图表不显示问题修复

### 问题诊断
用户反馈三个统计视图（基础统计、扩展统计、高级统计）的图表都不显示。

### 根本原因
1. **CSS `!important` 阻止视图显示**（核心问题）：`style.css` 第1053行 `.hidden { display: none !important; }` 的 `!important` 优先级高于 JavaScript 通过 `classList.toggle` 设置的内联样式。三个统计面板初始带有 `class="panel hidden"`，当用户切换视图时，JS 设置 `display:block` 被 `!important` 覆盖，视图面板始终隐藏。ECharts 在不可见容器（尺寸为0）中初始化，图表无法渲染。

2. **子菜单事件未初始化**：`initAdvancedSubMenu()` 从未被调用，高级统计子菜单（散点分布/大数定律/正态分布）切换无效。

3. **移动端交互缺失**：移动端底部导航（`.mobile-nav-btn`）和汉堡菜单（`#menuToggle`）未绑定事件。

### 修复内容

**`frontend/css/style.css`**：
- `.hidden { display: none !important; }` → `.hidden { display: none; }`

**`frontend/js/app.js`**：
- `DOMContentLoaded` 中增加 `initExtendSubMenu()`、`initAdvancedSubMenu()`、`initMobileNav()`、`initMenuToggle()` 调用
- 新增 `switchToView(view)` 统一视图切换逻辑（桌面端 `.view-btn` + 移动端 `.mobile-nav-btn` 共用）
- 新增 `initMobileNav()` 绑定移动端底部导航点击事件
- 新增 `initMenuToggle()` 绑定汉堡菜单和遮罩层事件
- 简化 `initViewToggle()`，复用 `switchToView()`
- `loadExtendView()` 中移除重复的 `initExtendSubMenu()` 调用
- `loadStats()` 中添加空状态和错误状态友好提示

**`frontend/index.html`**：
- 版本号更新为 `v=20260607f`
